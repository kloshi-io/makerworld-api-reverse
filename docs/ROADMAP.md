# Roadmap

## 0.1.x

- Stabilize resolver/downloader contracts.
- Expand fixture coverage for real-world payload variants.
- Improve diagnostics and warning detail.

## 0.2.x

- Add optional client factory for shared request config.
- Add first-class telemetry hooks for observability.
- Add fixture-based compatibility matrix by printer family.

## 0.3.x

- Add CLI wrapper for quick ad-hoc usage.
- Add richer type-level modeling of variant payloads.
- Add cache adapters for high-throughput ingest pipelines.

## Contribution Priorities

- Improve parser resilience against upstream payload drift.
- Add model URLs/payload fixtures from different locales.
- Add end-to-end tests for failure-mode recoverability.

## Good First Issue Checklist

- Choose issue labeled `good first issue`.
- Reproduce with a failing test first.
- Keep change scope focused.
- Update docs/examples if behavior changes.
- Add clear PR summary and migration note (if relevant).
